# Apple 日历订阅设计

## 目标

让用户把当前看板中仍有效的作业和考试订阅到自己的 Apple Calendar，由 iPhone、iPad 和 Mac 的系统日历负责展示和提醒。该功能不引入 AI、模型账号、共享额度或 Outlook/Google Calendar 产品入口。

## 非目标

- 不创建 iOS 原生应用，不使用 Apple 私有 API。
- 不从 Apple Calendar 回写完成状态；网页仍是唯一事实来源。
- 不在日历客户端轮询时启动浏览器、登录第三方平台或抓取外部 API。
- 不支持没有截止日期的事项。

## 用户流程

1. 已登录用户在设置中打开“Apple 日历”。
2. 用户生成私密订阅链接，并可复制链接或扫描网页生成的二维码。
3. 用户在 iPhone/iPad 的 Calendar 中添加订阅日历，并选择 iCloud 以同步到同一 Apple Account 的其他设备。
4. 该日历展示用户尚未完成、未隐藏、未删除且有截止时间的事项。
5. 用户仍在网页中完成、隐藏或删除事项；之后 Apple Calendar 在下一次订阅同步时更新或移除相应事件。
6. 用户可重置订阅链接；旧链接立即失效，需要重新订阅新链接。

## 数据与事件规则

### 数据来源

订阅源只读取当前用户的本地数据：Canvas、好课、智学盟、智慧树缓存及自定义待办。它复用每个平台的隐藏、标红和删除状态；标红不影响是否导出。

### 导出筛选

仅导出满足以下全部条件的事项：

- 有稳定平台/自定义事项 ID；
- 有可解析的截止日期或截止时间；
- 未完成（自定义待办）；
- 未隐藏、未删除（所有平台）。

外部平台缓存中的已过期事项不会导出。日期型截止事项导出为全天事件；带时间的截止事项导出为截止时刻附近的短事件。事件标题包含事项名称，描述包含来源、课程和原平台链接（如有）。

### 稳定同步

每个事项使用由“来源 + 原始 ID”派生的稳定 iCalendar UID。服务端保存每个用户已发布事件的轻量账本，记录 UID、内容指纹、序列号和最后出现时间。

- 内容或截止时间变化：保留 UID，递增 `SEQUENCE` 并更新事件。
- 事项不再满足导出条件：保留 UID，递增 `SEQUENCE`，输出 `STATUS:CANCELLED`。
- 取消事件保留有限窗口，确保常见订阅客户端有机会同步移除；窗口和清理策略在实现计划中明确为可测试常量。

## 接口与安全

- 提供无会话的 `GET` iCalendar 订阅端点；日历客户端无法携带网站会话。
- 链接携带 256-bit 随机 bearer token。数据文件只保存 token 哈希，绝不保存或显示旧 token 明文。
- 重置操作原子地更换 token，并使旧 token 返回 404。
- 订阅端点只返回 `text/calendar; charset=utf-8`，不泄露用户配置、凭据、平台 token 或其他账户数据。
- 订阅链接是访问凭证：仅在 HTTPS 启用后向用户开放；反向代理不得将完整订阅 URL 记录到访问日志。
- HTTP 缓存遵循 ETag/Last-Modified，避免客户端频繁获取相同内容，同时不得让共享缓存跨用户复用响应。

## Apple 提醒边界

Apple Calendar 负责系统通知。服务端可在 ICS 中提供默认提醒信息，但不承诺任何未在真实 iPhone 上验证过的提醒行为。若订阅日历不采纳服务端提醒信息，用户通过 Apple Calendar 的订阅日历提醒设置启用提醒。

验收必须在真实 iPhone 上依次验证：订阅成功、跨设备显示、新增事项、截止时间变更、完成后撤销、隐藏/删除后撤销，以及到点通知。

## UI

设置区新增一个 Apple Calendar 卡片：

- 未生成：生成订阅链接。
- 已生成：复制链接、显示二维码、显示安全提示、重置链接。
- 重置后：明确提示旧链接已失效，并提示用户在 Apple Calendar 中替换订阅。

不增加 Outlook、Google Calendar、AI Provider 或模型设置入口。

## 测试

自动化测试覆盖：

- 不同用户 token 与事件严格隔离；旧 token 在重置后失效。
- 五种来源的筛选、截止日期格式和标题/描述生成。
- 隐藏、删除、完成状态均不导出。
- UID 稳定性、时间更新的序列递增、取消事件生成与过期清理。
- 无缓存或无有效事项时输出合法空日历。
- 响应 Content-Type、缓存头、无敏感字段和无第三方抓取行为。

手工验收覆盖真实 iPhone 的订阅、同步、系统提醒和撤销事件显示。

## 成功标准

用户能在 Apple Calendar 中订阅自己的私密日历，并在手机上看到当前有效事项和系统提醒；用户在网页中完成、隐藏或删除事项后，该日历在后续同步中不再保留这些事项。所有账户数据与订阅 token 均保持隔离。
