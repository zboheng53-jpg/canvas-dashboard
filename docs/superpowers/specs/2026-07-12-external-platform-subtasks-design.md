# 外部平台作业子任务设计

## 背景

统一待办列表已支持自定义待办的可折叠子任务。Canvas、好课、智学盟和智慧树条目保留了相同位置的三角占位符，但尚不能操作。

## 目标

让四个外部平台的作业拥有与自定义待办一致的子任务体验：展开/收起、新增、勾选、编辑文本、设置截止日期和删除。按钮必须保持在现有 `item-subtask-slot`，不改变日期或右侧操作按钮的位置。

平台刷新后，作业只要以相同的平台来源和作业 ID 再次出现，就必须恢复原来的子任务。

## 非目标

- 不把子任务同步回任何第三方平台。
- 不改变外部作业本身的隐藏、标红或删除行为。
- 不持久化子任务面板的展开状态。
- 不修改现有自定义待办子任务的数据格式或 API。

## 数据模型与存储

新增每用户文件 `data/users/<username>/external_subtasks.json`，由一个专用存储模块管理，并复用 `storage.py` 的锁定更新和原子写入。

文件是以稳定键保存的对象：

```json
{
  "canvas:101": {
    "subtasks": [
      {"id": 1, "text": "阅读题目", "done": false, "due_date": null}
    ],
    "updated_at": "2026-07-12T12:00:00+08:00"
  },
  "zhixuemeng:work-42": {"subtasks": [], "updated_at": "2026-07-12T12:01:00+08:00"}
}
```

键由严格白名单来源 `canvas`、`haoke`、`zhixuemeng`、`zhihuishu` 和字符串化作业 ID 组成。因此各平台的同名或同 ID 作业不会相互覆盖，且外部刷新不会覆盖本地子任务。

## API 与数据流

四个现有 `/api/<platform>/todos` 路由在返回作业前，按来源和作业 ID 合并该作业的 `subtasks`；没有保存记录时返回空数组。前端的四条抓取路径无需额外请求即可获得子任务。

新增一个受当前登录会话保护的通用写入路由，接收来源、作业 ID 和完整 `subtasks` 数组。它只接受上述四个平台来源，原子地替换单个稳定键的子任务记录，并返回更新后的子任务。自定义待办继续使用现有 `/api/custom/todos/<id>` 写入路径。

子任务格式继续保持现有字段：`id`、`text`、`done`、可选 `due_date`。子任务完成状态不改变父作业的隐藏或完成状态。

## 前端交互与布局

统一列表项目继续携带 `source`、`rawId` 和 `subtasks`。展开状态从仅保存自定义 ID 改为保存来源和原始 ID 组成的稳定前端键。

渲染层对五类条目使用同一个可点击三角按钮和同一个子任务面板。原先外部条目的静态占位 `span` 变为按钮，但仍在 `item-subtask-slot` 内，因此网格位置、日期位置和桌面/移动端操作按钮位置不变。每一类条目均使用同样的子任务新增、勾选、编辑、截止日期和删除控件。

保存函数按来源分流：自定义待办继续请求既有 PUT 路由；外部作业请求新通用路由。写入失败时提示失败并重新抓取对应数据，防止浏览器内存长期与服务端不一致。

## 验证

后端测试覆盖：

1. 四个平台分别写入和读取子任务，且相同作业 ID 不串数据。
2. 各平台待办响应包含已保存子任务；无记录时包含空数组。
3. 使用相同来源和作业 ID 的后续抓取仍恢复子任务。
4. 非法来源、缺失作业 ID 或不合法写入返回请求错误。

Playwright 回归测试覆盖：

1. Canvas、好课、智学盟和智慧树各一条作业都可展开，完成完整的新增、编辑、勾选、截止日期和删除读写流程。
2. 刷新页面后子任务仍在且面板默认收起。
3. 外部条目的三角按钮在桌面和移动视口中仍位于原 `item-subtask-slot`，日期和右侧操作控件位置保持既有对齐约束。
4. 现有自定义待办子任务测试继续通过。
