# 待办时间分组与控制区调整设计

## 目标

在保留现有 Canvas Dashboard 简约视觉风格和功能行为的前提下，提高待办列表的信息组织效率：

- 按截止时间将任务分为“今天优先 / 本周安排 / 之后”；
- 增加来源筛选，快速查看某个平台的任务；
- 桌面端每个任务保持单行，四个操作位继续外露；
- 整理待办卡右上角，将平台关联与账户操作分开；
- 仅使用少量现有强调色，不重新设计顶部时间、天气和学期卡片。

## 已确认布局

顶部时间、日期、天气、学期信息及其卡片尺寸保持不变。待办主卡沿用现有白底、细边框、暖白页面背景和衬线标题。

待办主卡从上到下排列为：

1. 标题“待办清单”；
2. 右侧“关联平台”按钮与用户名菜单；
3. 自定义待办添加栏；
4. 左侧文字统计、右侧更新时间与刷新按钮；
5. 来源筛选标签；
6. “今天优先 / 本周安排 / 之后”三个时间分组及任务行。

不增加品牌栏、大号学业标题、学期进度条、大面积渐变或指标卡片。

## 来源筛选

来源筛选顺序固定为：

1. 全部
2. Canvas
3. 好课
4. 智学盟
5. 智慧树
6. 自定义

每个筛选标签显示该来源当前已渲染任务的数量，例如 `Canvas (3)`。数量包含现有列表中仍显示的完成或隐藏项，以确保标签数量与用户实际能看到的行数一致。

默认选择“全部”。选择某个来源后：

- 三个时间分组同时过滤；
- 没有匹配任务的分组隐藏；
- 没有任何匹配任务时显示“该来源暂无待办”；
- 不重新请求 API，不修改任务排序和持久化状态。

未选中的标签使用中性灰色；选中的标签只增加现有蓝色强调边框、文字色和极浅背景。

## 时间分组

分组基于页面已有的统一任务数组和 `window.customToday`：

### 今天优先

包含截止时间早于或等于今天结束时的未完成任务，因此逾期任务也进入“今天优先”。分组内继续使用现有紧急状态颜色区分逾期或临期任务。

### 本周安排

包含今天之后七个自然日内截止的任务。这里采用滚动七日窗口，而不是截至星期日，以匹配“接下来一周要安排什么”的使用场景。

### 之后

包含七日窗口之后截止的任务，以及没有截止日期的任务。

每个分组内继续使用现有排序规则：

1. 未完成或未隐藏项在前；
2. 有截止日期的项在前；
3. 截止时间升序；
4. 无截止日期项保持原始相对顺序。

完成或隐藏项仍按原有方式显示为弱化和删除线，不改变现有状态语义。

## 桌面任务行

在宽度大于 `768px` 时，每个任务使用严格的单行布局：

`来源标签 | 标题 | 课程或自定义标签 | 截止日期 | 四个操作位`

- 标题占用剩余宽度，过长时省略，不挤压右侧信息；
- 课程或标签、截止日期和操作位组成紧凑的右侧信息组；
- 日期纵向对齐；
- 四个操作位继续外露：子任务展开、标记重点、完成/隐藏、删除；
- 外部平台没有可展开子任务时继续显示禁用占位符，维持列对齐；
- hover 不改变行的 `margin` 或 `padding`，避免内容位移。

现有手机端紧凑交互保持不变：在 `768px` 及以下继续使用当前移动端更多操作菜单，以避免横向溢出和过小触控目标。本次只保证桌面任务行严格单行。

## 待办卡控制区

### 关联平台

原“系统登录”按钮改名为“关联平台”，继续展开现有 Canvas、好课、智学盟和智慧树连接卡片。

### 用户名菜单

用户名从静态文字变为按钮，点击后打开菜单：

- 开启 Apple Calendar 功能时显示“订阅苹果日历”；
- 始终显示“退出登录”；
- “订阅苹果日历”继续调用现有订阅弹窗和 API；
- “退出登录”继续调用现有退出逻辑。

点击菜单外部、按 `Escape` 或再次点击用户名均关闭菜单。打开用户名菜单时关闭关联平台面板；打开关联平台面板时关闭用户名菜单。按钮使用 `aria-expanded` 表示状态。

### 更新时间与刷新

“已更新 · 时间”和刷新按钮移到文字统计所在行的右侧。统计仍保持“待完成 N 项 / 即将到期 N 项”的当前文字形式，不改成指标卡片。

## 数据流与错误处理

所有平台 API、状态 API 和自定义待办 API 保持不变。渲染流程为：

1. 现有请求更新各平台数组；
2. `renderUnifiedList()` 构建统一任务数组；
3. 统一数组按现有规则排序；
4. 根据当前来源筛选过滤；
5. 根据截止时间分为三个时间组；
6. 渲染分组标题、计数和任务行；
7. 更新文字统计与来源标签计数。

某个平台加载失败时，继续渲染其他已成功来源，并沿用现有错误提示。缺失、无效或无法解析的截止时间进入“之后”，避免任务消失或渲染失败。

## 实现边界

正式改动限制在：

- `templates/index.html`
- `static/style.css`
- `tests/test_frontend_playwright.py`
- `tests/test_frontend_text_integrity.py`（仅在新增静态文字需要保护时）

不修改 Flask 路由、API 返回格式、数据文件、认证行为、任务 ID、排序语义或平台状态语义。不引入前端框架、图标库或第三方依赖。

## 验证

Playwright 回归覆盖：

- 桌面端来源筛选顺序、数量和激活状态；
- 选择来源后跨三个时间分组过滤，空分组隐藏；
- 逾期或今天任务进入“今天优先”；
- 未来七日任务进入“本周安排”；
- 更晚或无日期任务进入“之后”；
- 桌面任务标题、课程或标签、日期与四个操作位位于同一行；
- 长标题省略且右侧信息和按钮仍可见；
- “关联平台”继续展开平台卡片；
- 用户名菜单包含正确的 Apple Calendar 与退出入口；
- Apple Calendar 功能关闭时菜单不显示订阅入口；
- 更新时间和刷新按钮位于统计行；
- `375px`、`390px` 和 `768px` 下现有移动端操作菜单与无横向溢出行为不回退。

完成后运行相关 Playwright 测试和完整 `scripts/test.ps1`。若完整测试因环境依赖无法运行，必须报告具体未执行项和原因。

## 成功标准

用户打开页面后仍看到熟悉的原版简约界面，同时可以：

- 通过来源标签快速筛选任务；
- 按“今天优先 / 本周安排 / 之后”理解近期负荷；
- 在桌面端一行内读取任务并直接使用四个操作；
- 通过“关联平台”管理来源；
- 通过用户名菜单订阅 Apple Calendar 或退出登录。

新增布局不造成桌面任务行折行、hover 位移或手机端横向溢出。
